Conclusion

The prelim period gives insights into what being a DevOp means and familiarizes the skills needed to be one. We need these skills to approach different tactics for speeding up the software process, properly managing files, and enhancing the work in the system. In our prelim period, we also discussed the basics of SSH-based authentication and how to set up a GitHub. We are able to connect our virtual machines in GitHub so that we can push and commit changes to our repositories. We also have experience using CentOS, another type of open-source operating system like Ubuntu. We also learned the basics of creating an Ansible playbook. We were able to install programs for both Ubuntu and CentOS.

In our Midterm period, the challenges begin. We go deeper into the Ansible playbook and how we can utilize it for efficiency. We learn how to target specific nodes in the playbook. This way, we can provide control, precision, and efficiency in managing and configuring infrastructure using Ansible. Next, we are introduced to managing files and creating roles in the Ansible playbook. For me, this is the main topic that we used throughout the course. It is really important to learn roles and manage files because it makes it easier for an administrator to efficiently use these features for efficiency, to improve scalability, and effectiveness. To test our skills, we had to install programs that have prerequisites like files, tools, configurations, etc., in order for the installation to work. Roles and tasks make this possible and make the Ansible program easier.

In the final period comes other topics involving containerization, virtualization, Docker, and OpenStack. Containerization using Docker gives us experience in understanding the importance and portability of containers for software development and deployment. In OpenStack, I was introduced to installing Keystone, Glance, Nova, Neutron, Horizon, and Cinder setups that are known for cloud platforms. I know that all of these lessons will be crucial for my future and for my career. These lessons and courses are stepping stones to being a System Administrator, DevOps, and Cloud Engineer.
